# Auto-Fill branch_id Using Classes Table Relationship\n\n## 🎯 **Solution Overview**\n\nSince your `classes` table already has both `school_id` and `branch_id`, we can auto-fill the `branch_id` in the `subjects` table by joining on `class_code` and `school_id`.\n\n## 🔗 **Relationship Structure**\n\n```\nsubjects table          classes table\n┌─────────────────┐     ┌─────────────────┐\n│ subject_code    │     │ class_name      │\n│ subject         │     │ class_code      │ ← JOIN KEY\n│ school_id       │ ←───┤ school_id       │ ← JOIN KEY  \n│ class_code      │ ────┤ branch_id       │ ← SOURCE\n│ branch_id (NULL)│ ←───┤ section         │\n│ section         │     │ status          │\n└─────────────────┘     └─────────────────┘\n```\n\n## 🚀 **Quick Auto-Fill (Recommended)**\n\n### **Use `SIMPLE_AUTO_FILL_BRANCH_ID.sql`**\n\nThis is the fastest solution:\n\n```sql\n-- Main auto-fill query\nUPDATE subjects s\nINNER JOIN classes c ON s.class_code = c.class_code AND s.school_id = c.school_id\nSET s.branch_id = c.branch_id\nWHERE (s.branch_id IS NULL OR s.branch_id = '')\nAND c.status = 'Active';\n```\n\n## 📁 **Available Scripts**\n\n### 1. **`SIMPLE_AUTO_FILL_BRANCH_ID.sql`**
