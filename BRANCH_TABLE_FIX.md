# Branch Table Name Fix\n\n## 🐛 Issue Fixed\n\n**Error**: `Table 'elitedeploy.branches' doesn't exist`\n\n**Root Cause**: The branch validation code was trying to query a `branches` table that doesn't exist in the database.\n\n**Solution**: Updated the query to use the correct table name `school_locations` which is what stores branch information.\n\n## 🔧 Fix Applied\n\n### File: `elscholar-api/src/controllers/subjectsEnhanced.js`\n\n**Before:**\n```sql\nSELECT branch_id FROM branches WHERE branch_id = ? AND school_id = ? LIMIT 1\n```\n\n**After:**\n```sql\nSELECT branch_id FROM school_locations WHERE branch_id = ? AND school_id = ? LIMIT 1\n```\n\n## ✅ Result\n\n- ✅ Branch validation now works correctly\n- ✅ Admin users can access multiple branches with proper validation\n- ✅ Branch isolation enforced using the correct table\n- ✅ No more \"Table doesn't exist\" errors\n\n## 📝 Note\n\nIn this system:\n- **Branches** = **School Locations** \n- **Table name**: `school_locations`\n- **Purpose**: Store branch/location information for schools\n\nThe enhanced subjects controller now correctly validates branch access using the `school_locations` table.\n"
